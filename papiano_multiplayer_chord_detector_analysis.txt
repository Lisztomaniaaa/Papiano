ANALISIS PROFESIONAL — MASALAH CHORD DETECTOR LIVE MULTIPLAYER PAPIANO

File yang dianalisis: multiplayer (2).html
Topik: Chord detector antar player tidak muncul / tidak bekerja pada mode multiplayer

1. Ringkasan Masalah

Masalah utama bukan pada kemampuan chord detector membaca chord, melainkan pada jalur data multiplayer yang tidak masuk ke sistem chord detector secara konsisten.

Di file multiplayer, elemen tampilan untuk chord multiplayer sebenarnya sudah tersedia, yaitu container live chord multiplayer dan style untuk baris/chip chord pemain. Artinya, masalahnya bukan karena UI chord multiplayer belum ada.

Masalah terjadi karena note dari player lain diproses sebagai remote playback event. Event ini dipakai untuk audio, visual tuts, falling notes, dan efek pemain. Namun state chord remote tidak dipastikan masuk lebih dulu sebelum filter audio/volume/mute dijalankan.

Akibatnya, pada kondisi tertentu, event remote berhenti sebelum masuk ke chord detector multiplayer. Hasilnya: pemain lain tetap bisa terlihat bermain atau event tetap masuk, tetapi chord antar player tidak muncul.

2. Perbedaan Jalur Local Chord dan Remote Chord

Pada pemain lokal, chord detector bekerja karena nada yang ditekan masuk ke state lokal seperti pressed notes / pressStartTimes / active local notes. Setelah itu sistem memanggil update chord display.

Alur lokal:
Player lokal menekan tuts
→ nada masuk ke state lokal
→ chord detector membaca state lokal
→ chord lokal muncul di display

Pada multiplayer, nada player lain masuk sebagai remote note event dari realtime database.

Alur remote saat ini:
Player lain menekan tuts
→ event masuk dari realtime database
→ diproses sebagai remote note
→ masuk audio/visual remote
→ chord remote belum tentu ikut diproses

Ini membuat chord detector antar player tidak stabil atau tidak muncul sama sekali.

3. Akar Masalah Utama

Akar masalah paling penting adalah chord detector remote terlalu bergantung pada jalur audio/visual remote.

Remote chord state seharusnya diperbarui segera setelah event noteOn/noteOff valid diterima. Namun pada struktur saat ini, pemrosesan chord remote bisa terhalang oleh pengecekan seperti mute, remote volume, playback gain, atau kondisi audio lain.

Masalahnya:
Chord detection ikut mati jika remote audio dimute, volume remote nol, atau event dihentikan oleh filter playback.

Padahal chord detection secara konsep harus bergantung pada event nada, bukan pada audio playback.

Kesimpulan teknis:
Chord detector multiplayer tidak boleh berada setelah return condition milik audio/visual playback.

4. Kenapa Ini Membuat Chord Antar Player Tidak Muncul

Jika remote event melewati jalur seperti ini:

remote note event diterima
→ cek player muted / remote volume / playback gain
→ return lebih awal
→ chord state tidak diperbarui

maka chord display multiplayer tidak punya data aktif untuk ditampilkan.

Dari sisi pengguna, ini terlihat seperti:
- player lain bisa masuk room,
- player lain bisa menekan tuts,
- beberapa visual/audio mungkin tetap terlihat pada kondisi tertentu,
- tetapi chord antar player tidak muncul.

Penyebabnya bukan karena chord detector gagal membaca harmoni, tetapi karena chord detector tidak menerima active note state milik player remote secara konsisten.

5. Masalah Tambahan yang Perlu Dicatat

5.1 Self Player Tidak Ditampilkan di Multiplayer Chord Strip

Pada umumnya sistem chord multiplayer mengabaikan playerId milik diri sendiri agar chord lokal tetap tampil di chord display utama.

Ini benar secara desain, tetapi bisa menipu saat testing.

Jika pengujian dilakukan dengan akun yang sama, tab yang sama, atau playerId yang dianggap sama dengan mpSelfId, maka chord multiplayer tidak akan muncul karena sistem menganggap itu bukan remote player.

5.2 Minimal Note Logic

1 nada bukan chord.
2 nada juga bukan chord normal.

Jika sistem saat ini mencoba membaca 2 nada sebagai chord penuh, itu rawan menghasilkan fallacy musikal. Dua nada hanya boleh dibaca sebagai interval/dyad jika Beta Mode aktif.

Aturan yang benar:
- 1 nada: tidak tampil sebagai chord.
- 2 nada: tidak tampil sebagai chord normal.
- 2 nada hanya boleh tampil sebagai interval/dyad di Beta Mode.
- 3 nada atau lebih: masuk ke chord detector utama.

Namun ini bukan penyebab utama chord tidak muncul. Ini adalah masalah akurasi musikal yang perlu dijaga saat memperbaiki sistem.

5.3 Stuck Note Karena noteOff Hilang

Remote note bergantung pada noteOn dan noteOff. Jika noteOff hilang karena lag, disconnect, atau event realtime terlambat, active note remote bisa tertinggal.

Jika active note tidak dibersihkan, chord bisa stuck atau hasil chord menjadi salah.

Diperlukan fail-safe cleanup berbasis timestamp untuk membersihkan note remote yang terlalu lama aktif.

6. Diagnosis Final

Diagnosis utama:
Chord detector antar player tidak muncul karena update state chord remote tidak ditempatkan pada jalur event note yang paling aman.

Seharusnya:
remote note event valid
→ update remote chord state
→ baru proses audio/visual/mute/volume

Bukan:
remote note event valid
→ cek audio/mute/volume
→ mungkin return
→ chord state tidak pernah diperbarui

Dengan kata lain, chord detector multiplayer saat ini terlalu terikat pada remote playback, padahal seharusnya terikat pada remote note event.

7. Rekomendasi Perbaikan Targeted

Perbaikan tidak perlu rewrite besar.
Perbaikan cukup targeted pada jalur event remote.

Yang perlu dilakukan:

1. Pindahkan pemanggilan remote chord noteOn/noteOff ke bagian paling awal setelah event remote valid diterima.

2. Jangan letakkan update chord remote setelah pengecekan:
- muteInstrument,
- remote playback gain,
- volume remote,
- soundfont readiness,
- visual rendering condition,
- audio playback condition.

3. Pastikan chord detection tetap berjalan walaupun:
- audio remote dimute,
- volume remote nol,
- visual remote dimatikan,
- soundfont remote belum siap.

4. Chord detection harus membaca event musikal, bukan hasil audio playback.

5. Tetap pisahkan chord lokal dan chord remote.

6. Jangan gabungkan semua player menjadi satu chord global.

7. Gunakan active note state per player.

8. Tambahkan fail-safe cleanup untuk remote note yang tidak menerima noteOff.

8. Struktur Logika yang Benar

Alur noteOn remote yang benar:

remote event diterima
→ validasi room/player/event
→ jika event dari player lain, update activeNotes milik player itu
→ jadwalkan update chord display multiplayer
→ setelah itu baru proses audio dan visual remote

Alur noteOff remote yang benar:

remote noteOff diterima
→ validasi room/player/event
→ hapus note dari activeNotes milik player itu
→ jadwalkan update chord display multiplayer
→ setelah itu baru proses release audio/visual remote

9. Acceptance Test Setelah Perbaikan

Perbaikan dianggap benar jika:

1. Chord lokal tetap muncul normal.
2. Chord milik remote player muncul saat remote player memainkan minimal 3 nada valid.
3. Chord remote tetap muncul walaupun audio remote dimute.
4. Chord remote tetap muncul walaupun volume remote nol.
5. Chord tidak tercampur antar player.
6. Player A dan Player B punya hasil chord masing-masing.
7. 1 nada tidak disebut chord.
8. 2 nada tidak disebut chord normal kecuali Beta/Dyad Mode aktif.
9. Jika player disconnect, chord player tersebut hilang.
10. Jika noteOff hilang, chord tidak stuck selamanya.
11. Tidak ada console error.
12. Tidak ada perubahan layout piano, hitbox, audio engine, atau falling notes alignment.

10. Kesimpulan

Masalah chord detector live antar player bukan karena UI belum ada dan bukan karena chord detector utama rusak.

Masalahnya adalah penempatan hook chord remote yang salah. Chord state remote harus diperbarui dari event note multiplayer, bukan dari jalur audio/visual yang bisa berhenti karena mute, volume, atau kondisi playback.

Solusi paling aman adalah memindahkan update chord remote ke awal handler remote noteOn/noteOff, sebelum semua filter audio dan visual dijalankan.
